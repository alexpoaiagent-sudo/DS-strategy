# РП#21: Система мониторинга агентов + механизм принудительного согласования

## Проблема

**Проблема #1: Обход ритуала согласования**
- Агент регулярно начинает работу БЕЗ прохождения ритуала согласования
- Правило уже существует в MEMORY.md (правило #6), но игнорируется
- Пользователь вынужден останавливать агента вручную каждую сессию

**Проблема #2: Падение агентов без уведомлений**
- Агенты (strategist, extractor, scheduler) могут упасть без уведомления
- Пример: OAuth 401 ошибки 19 марта — пользователь узнал только при ручной проверке
- Нет автоматической системы мониторинга и алертов

## Цель

1. **Механизм принудительного согласования:** Сделать невозможным обход ритуала согласования
2. **Система мониторинга:** Автоматическая проверка агентов + обязательные уведомления при падении

## Артефакты

### 1. Механизм принудительного согласования

**1.1. MEMORY.md — усиленное правило #0**
- Переместить правило согласования на позицию #0 (первое в секции БЛОКИРУЮЩИЕ)
- Усилить формулировку с явным алгоритмом действий
- Добавить примеры нарушений и последствия
- Формулировка: "Это правило НЕ МОЖЕТ быть проигнорировано ни при каких обстоятельствах"

**1.2. protocol-open.md — чеклист**
- Добавить явный чеклист с шагами которые нельзя пропустить
- Каждый шаг должен быть выполнен явно
- Связать с правилом #0 в MEMORY.md

### 2. Система мониторинга агентов

**2.1. health-check.sh**
- Проверка статуса агентов через launchctl
- Проверка последних логов на ошибки (OAuth 401, API errors)
- Проверка времени последнего успешного запуска
- Выход с кодом 0 (OK) или 1 (ERROR)

**2.2. Алерты**
- macOS notification при ошибке
- Telegram уведомление (если настроено)
- Лог ошибок в ~/logs/health-check/

**2.3. com.exocortex.health-check.plist**
- launchd агент для автоматического запуска health-check.sh
- Расписание: каждый час
- Запуск при загрузке системы

**2.4. Документация**
- Обновить project-brain.md с информацией о health-check
- Добавить инструкции по настройке Telegram алертов
